# План рефакторинга модуля Payday 2

Ниже представлен поэтапный план рефакторинга, от самых быстрых и критичных задач до более глубоких архитектурных изменений. Логика работы приложения не меняется, только улучшается безопасность, скорость и поддерживаемость кода.

## Этап 1: Критичные уязвимости и баги (Быстро, Высокий приоритет)
На этом этапе мы закрываем дыры в безопасности и чиним сломанный код, не трогая сложную логику.

- [x] **1.1. Исправление `create_transfer.php`**
  - Устранить синтаксическую ошибку (`Parse error: Unmatched '}' on line 140`).
  - Проверить, используется ли файл как fallback. Если да — починить POST-логику, если нет — корректно удалить, перенеся остатки логики в AJAX-контроллер.
- [x] **1.2. Закрытие прямого доступа к обработчикам**
  - Добавить строгую проверку прав `veranda_require('payday')` в самое начало файлов `ajax.php` и `post.php`.
  - Убедиться, что скрипты корректно завершаются (HTTP 403), если пользователь обращается к ним напрямую без нужного права.
- [x] **1.3. Внедрение CSRF-защиты**
  - Сгенерировать CSRF-токен в сессии (если еще нет глобального) и вывести его в `view.php` (например, в `window.PAYDAY_CONFIG` или `<meta>` тег).
  - Настроить JS (`payday2.js`) на автоматическое добавление токена во все POST/AJAX-запросы.
  - Добавить проверку токена в `ajax.php` и `post.php` перед выполнением любых изменяющих действий (создание переводов, удаление, изменение связей).

## Этап 2: Вынос хардкода и настройки (Средняя сложность)
Делаем модуль переносимым и настраиваемым без изменения PHP-кода.

- [x] **2.1. Создание локального конфигуратора**
  - Создать файл `payday2/local_config.json` (или `config_local.php`), в который вынести зашитые значения: 
    - `telegram_chat_id`, `telegram_message_thread_id`
    - `service_user_id` (сейчас `4`)
    - `account_ids` (сейчас `1, 8, 9`)
- [x] **2.2. UI для управления настройками**
  - Добавить кнопку "шестеренки" в шапку или меню `view.php`.
  - Сверстать модальное окно с формой для редактирования параметров конфига.
- [x] **2.3. Обработчики сохранения конфига**
  - Добавить AJAX-эндпоинт для безопасного сохранения новых настроек в локальный файл.
  - Заменить все захардкоженные значения в `ajax.php` и `view.php` на вызовы из нового конфига.

## Этап 3: Оптимизация бэкенда и запросов (Средняя сложность)
Ускоряем загрузку и синхронизацию, исправляем N+1 и логические несостыковки.

- [x] **3.1. Ускорение IMAP-поиска (Ветка OUT)**
  - Убрать повторное чтение `.env` внутри эндпоинта `ajax.php`.
  - Добавить в критерии поиска IMAP параметр `BEFORE` вместе с `SINCE` и `FROM`, чтобы сервер почты фильтровал письма строгим диапазоном, как принято в проекте.
- [ ] **3.2. Оптимизация рендера и устранение N+1 во `view.php`**
  - Вынести вызов Poster API для получения номеров столов **до** цикла вывода таблицы.
  - Сохранить загрузку финансовых транзакций (проверка транзакций из БД) при старте, чтобы страница отображала актуальные данные, но выполнять ее пакетно (одним SQL-запросом с `IN (...)`), а не в цикле.
- [ ] **3.3. Корректировка функции "Обнулить" (Сброс дня)**
  - Изменить текст кнопки и confirm-диалога на "Сбросить день (Soft Reset)", чтобы было понятно, что связи физически не удаляются.
  - Убедиться, что логика обновления данных (при следующем синке) использует `INSERT ... ON DUPLICATE KEY UPDATE` или аналогичный механизм: если запись приходит снова, она должна снимать флаг `was_deleted = 1` и восстанавливаться, сохраняя привязку по ID.

## Этап 4: Рефакторинг фронтенда (Высокая сложность)
Очищаем клиентский код от "костылей" и хрупких паттернов.

- [ ] **4.1. Отказ от агрессивного PJAX**
  - Переписать логику `payday2.js`: убрать использование `eval()` для подхвата конфигурации.
  - Отказаться от глобальной перезаписи `document.addEventListener` и `window.addEventListener`.
  - Заменить полную перерисовку `innerHTML` рабочей зоны на точечное обновление нужных таблиц/контейнеров или использование легкого стейт-менеджера.
- [ ] **4.2. Разделение слоев (Очистка View)**
  - Вынести все inline-стили (`<style>`, `style="..."`) из `view.php` в `payday2.css`.
  - Вынести все inline-скрипты (`onclick="..."`, `<script>`) в `payday2.js`, используя делегирование событий.
