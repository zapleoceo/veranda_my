## ТехническийДолг

Ниже список доработок, которые делались поверх текущей версии и были откатаны назад для проверки. Их имеет смысл вернуть/перенести аккуратно, по одной, с проверкой на проде.

### TableReservation / Tr2 (схема бронирования)

- Источник “занято сейчас” должен быть не эвристика “не в freeTables”, а открытые чеки из `dash.getTransactions` со `status=1` (гости ещё сидят). Требуется маппинг `table_id -> table_num` через `spots.getTableHallTables`.
- Показ подписей броней на столах:
  - получать брони через `incomingOrders.getReservations`,
  - собирать интервалы `date_start/date_end` по каждому `table_num`,
  - отображать на столе 1–2 ближайших интервала (пример `18:00-22:00 · 19:30-21:30`).
- Получение броней “на день” должно быть устойчивым к особенностям Poster:
  - пробовать `getReservations(date_from/date_to)` и `timezone=client`,
  - при пустом ответе — fallback: `getReservations` без диапазона + фильтрация по `date_reservation` на стороне сайта.
- Маппинг столов: если `getReservations` не отдает `table_id` для конкретной брони, нужно подтягивать детали через `incomingOrders.getReservation(incoming_order_id)` и извлекать `table_id` оттуда.
- UI: в модалке заявки на бронь дата в шапке должна заполняться (не пусто), `reqStart` должен корректно отображать выбранный старт.
- Кеширование фронта: при изменениях скриптов нужно обновлять query-параметр `?v=...` у подключаемого JS, чтобы пользователи не оставались на старом кеше.

### reservations.php (Kitchen Analytics)

- Время на странице броней должно показываться в таймзоне заведения (Вьетнам), а не в таймзоне сервера:
  - заменить `date()/strtotime()` на `DateTimeImmutable` с `POSTER_SPOT_TIMEZONE` (по умолчанию `Asia/Ho_Chi_Minh`).
- Для броней Poster:
  - корректно обрабатывать `status` (0 — новый, 1 — принят, 7 — отменён),
  - показывать интервал `start–end` (end вычислять через `duration` из Poster).

### Poster API / инфраструктура

- Для локальной диагностики на Windows иногда нужен режим `curl` без проверки SSL-цепочки (не включать на проде): флаг окружения наподобие `POSTER_CURL_INSECURE=1`.
